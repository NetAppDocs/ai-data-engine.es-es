= Configura y configura AI Data Engine
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Como administrador de almacenamiento de ONTAP, tú usas ONTAP System Manager para integrar AI Data Engine (AIDE) con tu entorno AFX. Después de que hayas completado la instalación de tus nodos de cómputo de datos, ONTAP puede descubrirlos y agregarlos a un clúster. Puedes agregar nodos AIDE durante la configuración inicial del clúster AFX o agregarlos a un clúster AFX existente, y luego completar la configuración requerida de AIDE.

.Antes de empezar
* Tienes NetApp DCNs o servidores de terceros instalados y conectados a los switches del clúster. Consulta la documentación de instalación de link:../install-aide-server/install-workflow.html["Software AIDE en servidores de terceros"] o link:../install-dcn/install-workflow.html["DCN de NetApp"] para más información.
* (Para servidores de terceros) Has completado link:../install-setup/install-aide-as-software.html["Instalación del software AIDE"] en cada servidor. ONTAP puede entonces descubrir estos servidores como nodos de computación, y puedes añadirlos a un clúster.




== Configura AIDE e intégralo con un sistema de almacenamiento AFX

Hay dos formas de configurar AIDE e integrarlo con un sistema de almacenamiento AFX. Elige la opción que sea más adecuada para tu entorno.



=== Configura AIDE con un nuevo clúster AFX

Si estás configurando AIDE como parte de un despliegue inicial del sistema AFX, el proceso de configuración de AIDE forma parte del flujo de trabajo estándar de configuración del clúster AFX. Durante la configuración del clúster ONTAP, System Manager descubre automáticamente los nodos de computación conectados y los incluye como parte de la configuración del clúster.


NOTE: Necesitarás link:../install-setup/configure-oidc.html["configurar la autenticación de OpenID Connect"] para habilitar el acceso a AIDE Console.



=== Configura AIDE con un clúster AFX existente

Si vas a añadir nodos de cómputo (ya sean NetApp DCN o servidores de terceros) a un sistema de almacenamiento AFX existente, necesitas añadir los nodos y terminar de configurar tu clúster AIDE.



==== Agrega nodos de cómputo al clúster

ONTAP detecta los nodos de computación conectados a tu red de clúster y los muestra en System Manager. link:../administer/dcn-add-node.html["Agrega nodos de computación a tu clúster de almacenamiento para convertirlo en un clúster AIDE"].



==== Termina de configurar tu cluster AIDE

Después de añadir los nodos de computación y completar la configuración inicial de la red, puedes terminar de configurar tu nuevo clúster AIDE.

.Acerca de esta tarea
En la página se muestra una ventana de guía de configuración de bienvenida con los pasos de configuración necesarios. Se muestra una marca de verificación a la izquierda de cada paso o elemento de acción que ya se ha completado. Se muestra una flecha (image:icon_arrow.gif["flecha azul apuntando a la derecha"]) para los elementos que aún no se han completado. Para cada una de las acciones incompletas:

* Selecciona el enlace del título habilitado y completa la acción de configuración para tu entorno.
* Si el enlace está desactivado, pasa el ratón por encima del título para mostrar la acción necesaria para activarlo y continuar con la acción de configuración.


Cada paso tiene un enlace a documentación adicional cuando es posible. Parte de la configuración podría incluirse como parte de la configuración del cluster AFX en https://docs.netapp.com/us-en/ontap-afx/["Documentación de AFX"^].

.Pasos
. Selecciona *Configurar grupos de agregación de enlaces y VLAN*.
+
Si es necesario para tu entorno, puedes configurar LAG y VLAN a nivel de clúster. Al seleccionar esta opción, se abre la página de puertos Ethernet.

. Selecciona *Configurar protocolos de red*.
+
Puedes configurar los protocolos de red de la máquina virtual de almacenamiento (SVM).

. Selecciona *Actualizar software de Data Compute Node*.
+
Si hay una imagen de software AIDE disponible para actualización, esta opción estará activada. Selecciona esta opción para abrir la página de actualización de software. Para más información, consulta link:../administer/update-aide-firmware-software.html["Actualizar el software AIDE"].

. Selecciona *Configurar la red del motor de datos*.
+
Esta opción muestra la página para configurar la dirección de red del data engine.

. Selecciona *Crear interfaces de red intercluster*
+
Esta opción inicia la página para configurar las interfaces de red intercluster como parte de un procedimiento de dos pasos si tienes intención de catalogar datos remotos.

. Selecciona *Peer with other ONTAP storage systems*.
+
Después de crear las interfaces de red intercluster, puedes establecer relaciones de peering con otros sistemas de almacenamiento ONTAP.

. Selecciona *Añadir un contenedor de datos*.
+
Puedes añadir un volumen para que lo use AIDE y asociarlo a una SVM específica.

. Selecciona *Add Workspaces*.
+
Esto abre la página de espacios de trabajo de Data Engine donde puedes link:../workspaces/create-workspace.html["crear un espacio de trabajo"].

. Selecciona *Configurar OpenID Connect*.
+
Necesitarás link:../install-setup/configure-oidc.html["configura la autenticación OpenID"] para habilitar el acceso a AIDE Console.



.Después de que termines
Después de que se completen todos los elementos de acción, la ventana de orientación se oculta. Puedes volver a abrirla y cerrarla manualmente cuando lo necesites.



== Activa tu licencia AIDE para implementaciones de DCN en NetApp

Si tienes NetApp DCNs en tu configuración, necesitas link:../install-setup/install-license-task.html["instala tu licencia de servicios premium de AIDE"] después de que tu cluster AIDE esté configurado e integrado con el sistema de almacenamiento AFX. La licencia de servicios premium de AIDE es necesaria para la funcionalidad completa de AIDE, incluyendo las capacidades de vectorización y guardrails.
