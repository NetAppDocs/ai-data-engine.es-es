= Configura OpenID Connect para AIDE en ONTAP
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Como administrador de un clúster ONTAP, puedes usar ONTAP System Manager para configurar la autenticación OpenID Connect (OIDC) para un clúster AI Data Engine (AIDE). Esto te da un inicio de sesión seguro y centralizado a través de un proveedor de identidades externo (IdP).


CAUTION: Debes configurar OIDC para acceder a la AIDE Console. Cuando está configurado, toda la autenticación fluye a través de OIDC. Si OIDC no está configurado, la consola no estará disponible para ningún usuario (administradores, ingenieros de datos y científicos de datos). System Manager usará la autenticación local de ONTAP cuando OIDC no esté configurado.

Ten en cuenta también lo siguiente sobre la configuración de OIDC para el acceso a AIDE:

* A grandes rasgos, la configuración de OIDC consta de dos pasos principales. Primero necesitas realizar la configuración básica usando System Manager. Luego puedes completar la configuración asignando roles externos mediante la CLI de ONTAP.
* No puedes modificar una configuración OIDC existente. Si necesitas hacer un cambio, primero elimina la configuración y crea una nueva con los ajustes que quieras.
* Si desactivas o eliminas OIDC, System Manager volverá a la autenticación local de usuarios de ONTAP.




== Descripción general de OIDC

OpenID Connect (OIDC) es un protocolo de autenticación construido sobre el marco OAuth 2.0. Extiende OAuth 2.0, que se usa principalmente para autorización, añadiendo una capa de identidad. OIDC introduce el concepto de un ID token, que es un JSON Web Token (JWT) que contiene claims sobre el evento de autenticación y la identidad del usuario.

Necesitas seleccionar y configurar un proveedor de identidad externo (IdP) compatible con AFX con AIDE. El IdP autentica a los usuarios y emite tokens que AFX, a través de System Manager, puede usar para otorgar acceso a AIDE Console.



== Configura proveedores de identidad de terceros

Para autenticarse mediante OIDC, primero tienes que configurar un IdP externo. La implementación de ONTAP de OIDC utiliza reclamos de rol en los tokens para aplicar RBAC. Cuando configures un IdP, asegúrate de que esté configurado para devolver reclamos de rol en el id token y el access token. ONTAP admite dos IdPs para la autenticación de OIDC: Entra ID y Active Directory Federation Services (ADFS).



=== Entra ID

Puedes configurar Entra ID usando los siguientes pasos a grandes rasgos:

. Crea un nuevo registro de aplicación en la página de configuración de Entra ID.
. Establece el valor de Redirect URI (Web) en  `\https://$CLUSTER_MGMT_IP/oidc/callback`, sustituyendo la dirección IP de gestión del clúster o el FQDN correspondiente.
. Crea los roles necesarios en App Roles y asígnalos a tus usuarios.
. Actualiza las reclamaciones de token en Configuración de token para devolver roles en id-token y access-token.


Consulta https://learn.microsoft.com/en-us/power-pages/security/authentication/openid-settings["Configura un proveedor de OpenID Connect con Entra ID"^] para más información.



=== Active Directory Federation Services

Puedes configurar AD FS usando los siguientes pasos a grandes rasgos:

. Crea un nuevo grupo de aplicaciones y selecciona *Aplicación de servidor que accede a una API web*.
. Establece el valor de Redirect URI (Web) en  `\https://$CLUSTER_MGMT_IP/oidc/callback`, sustituyendo la dirección IP de gestión del clúster o el FQDN correspondiente.
. Configura las reclamaciones para devolver roles en los tokens.


Consulta https://learn.microsoft.com/en-us/azure/active-directory-b2c/identity-provider-adfs?pivots=b2c-user-flow["Agrega AD FS como proveedor de identidad de OpenID Connect"^] para más información.



== Configura OIDC en System Manager

Después de configurar tu IdP, puedes configurar la autenticación OIDC en System Manager para permitir el acceso seguro a AIDE Console.


TIP: Puedes proporcionar el URI de metadatos para rellenar automáticamente los campos de configuración de OIDC en System Manager. Revisa la documentación de tu IdP para obtener el formato exacto del URI.

.Antes de empezar
* Necesitas tener acceso de administrador a System Manager.
* Tu proveedor de identidad OIDC debe estar configurado y accesible.


.Pasos
. En System Manager, selecciona *Cluster* y luego *Settings*; localiza la tarjeta OpenID Connect.
. Si OIDC ya está configurado, puedes editar o desactivar la configuración. Si OIDC no está configurado, selecciona image:icon_gear_white_bg.png["icono de engranaje"] para empezar el proceso de configuración.
. En Configurar OpenID Connect, proporciona valores para los siguientes campos:
+
** Proveedor
** Emisor
** URI del conjunto de claves web JSON
** Punto final de autorización
** Punto final de token
** Punto final de la sesión
** Emisor del token de acceso (opcional)


. En configuración del cliente, indica los valores de los siguientes campos:
+
** ID de cliente
** Reclamación de usuario remoto
** Intervalo de actualización


. En Detalles de la conexión, proporciona valores para los siguientes campos:
+
** Dirección IP del clúster o FQDN
** Proxy de salida (opcional)


. En Asignación de funciones externa, selecciona una asignación de funciones existente o define una nueva función para el usuario de ONTAP `admin`.
. Selecciona *Enable now* y luego *Save*. System Manager se actualizará para aplicar la nueva configuración de autenticación.
. Inicia sesión con tus credenciales de IdP; después de autenticarte correctamente, volverás a System Manager.


.Después de que termines
Completa la configuración de OIDC configurando la asignación de roles externos.



== Configura la asignación de roles externos usando la CLI

La asignación de roles externos es una función de ONTAP que te permite asignar los roles de IdP externos a los roles de ONTAP correspondientes. Necesitas configurar esta asignación para asegurarte de que los usuarios que se autentican a través de OIDC reciban los permisos RBAC apropiados en ONTAP.

.Acerca de esta tarea
Esta tarea asigna el ingeniero de datos y el científico de datos a los roles de ONTAP correspondientes. Necesitas actualizar los ejemplos de comandos con los nombres de rol adecuados según tu entorno. Ten en cuenta que ya deberías haber asignado el rol de administrador de almacenamiento al rol de ONTAP `admin` durante la configuración básica de OIDC en System Manager.

.Pasos
. Usando SSH, inicia sesión en el CLI de ONTAP con una cuenta que tenga privilegios administrativos.
. Configura la asignación de roles para el rol de data engineer; por ejemplo:
+
[source, bash]
----
security login external-role-mapping create -external-role dataEngineer -provider entra -ontap-role data-engineer
----
. Configura la asignación de roles para el rol de científico de datos; por ejemplo:
+
[source, bash]
----
security login external-role-mapping create -external-role dataScientist -provider entra -ontap-role data-scientist
----
